# Phase-by-Phase Guides

Use this section when you know which step of discovery you are in and need practical instructions for running it.

Each phase page tells you:
- when to use the phase,
- who should be involved,
- what to cover,
- what "done" looks like,
- and what usually comes next.


> Note: Not every phase is “free by default”. See the [commercial boundary guide]({{ "/commercial-boundary/" | relative_url }}) for when to keep discovery directional vs when to move into paid discovery.

## Phases

| Phase | Use it for | Primary lead |
|---|---|---|
| [Initial discovery intake](intake/) | Gather enough context before the first serious customer call | Sales / Account Executive |
| [Sales discovery call](sales-discovery/) | Test whether the opportunity is real, urgent, and worth deeper discovery | Sales / Account Executive |
| [Workflow workshop](workflow-workshop/) | Turn feature requests into a real workflow, key exceptions, and an MVP slice | Product Manager / Delivery Lead |
| [Technical deep dive + NFR review](technical-deep-dive/) | Validate integrations, environments, access, and measurable quality targets | Solutions Architect |
| [Security / procurement track](security-procurement/) | Surface onboarding blockers early and organize required evidence | Security / Compliance / GRC |
| [Prototype / Proof of Value (PoV)](pov/) | Test the riskiest assumptions with a thin end-to-end slice | Product Manager / Delivery Lead + Engineering + SA |
| [Handover to delivery](handover/) | Turn discovery outputs into a delivery-ready packet and backlog starting point | Product Manager / Delivery Lead |
