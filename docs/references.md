# References (Optional reading)

These are sources and frameworks commonly used when building a disciplined requirements discovery process.

- ISO/IEC/IEEE 29148 — Requirements engineering processes and products
- BABOK (IIBA) — Elicitation & collaboration, requirements lifecycle management
- Scrum Guide — Backlog refinement as an ongoing activity
- SEI — Quality Attribute Workshop (QAW) and ATAM (architecture tradeoffs)
- User Story Mapping (Jeff Patton) — Workflow mapping and release slicing
- Use-Case 2.0 (Ivar Jacobson) — Use-case slices as backlog items
- EventStorming — Collaborative domain/process modeling
- Shared Assessments SIG, CSA CAIQ, SOC 2, NIST 800-53, OWASP ASVS — Common security/procurement artifacts
